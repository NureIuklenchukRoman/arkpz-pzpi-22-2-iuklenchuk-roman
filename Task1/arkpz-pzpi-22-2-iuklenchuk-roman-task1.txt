Хід роботи
У рамках розробки програмної системи для оренди маленьких складів було здійснено низку етапів роботи, починаючи від аналізу вимог і до впровадження функціональних можливостей системи.

ВИСНОВКИ
У результаті виконаної роботи була розроблена програмна система для оренди маленьких складів, що задовольняє основні потреби користувачів: зручне бронювання, швидка оплата та інтеграція з платіжними системами. Система успішно автоматизує процеси оренди складів, що дозволяє зменшити навантаження на адміністрацію та забезпечити комфорт користувачам.
















Додаток А
Vision and Scope Document
1. Передумови
1.1. Актуальність проблеми
     У сучасному світі багато людей стикаються з проблемою зберігання своїх речей через обмежений простір у домі або офісі. Це може бути зумовлено переїздами, сезонними змінами або просто потребою в тимчасовому звільненні місця. Часто виникає потреба у зберіганні речей на складі, але знайти доступне і зручне місце для оренди може бути складно.
     Програмна система для оренди маленьких складів пропонує рішення цієї проблеми. Вона дозволяє користувачам легко знаходити, бронювати і оплачувати склади, забезпечуючи зручний доступ до цієї послуги без зайвих труднощів. Система автоматизує процеси бронювання та управління орендою складів, що зменшує час і зусилля, необхідні для пошуку і організації зберігання речей.
1.2. Аналіз аналогічних рішень
Neighbor

Public Storage
Це  одна з найбільших компаній, що надає послуги з оренди складів в США. Платформа дозволяє користувачам знайти, забронювати і оплатити склад через інтернет. Користувачі можуть вибирати між різними типами складів, від малих до великих.
Переваги:
    • Велика кількість складів в різних регіонах.
    • Високий рівень безпеки на складах.
    • Наявність додаткових послуг, таких як продаж пакувальних матеріалів.
Недоліки:
    • Висока вартість оренди порівняно з іншими варіантами.
    • Обмежений доступ до складів у нічний час.

1.3. Способи монетизації
• Плата за оренду: Основне джерело доходу — комісія з кожної оренди складу. Користувачі платять за послугу оренди складу, а платформа отримує частку від цієї транзакції.
• Додаткові послуги: Можливість надання додаткових послуг, таких як охорона, клімат-контроль, пакувальні матеріали, доступ до транспорту для перевезення речей тощо. Ці послуги можуть бути надані за додаткову плату, що збільшує загальний дохід.
• Платні підвищення: Пропозиція платних підписок для орендодавців, що дозволяють отримати додаткові функції, такі як пріоритетна позиція в результатах пошуку, аналітика використання складу або більша видимість серед користувачів.

2. Vision of the Solution
2.1. Призначення системи
Система буде надавати доступ до широкого асортименту складських приміщень різних розмірів і умов, щоб задовольнити потреби користувачів у різних ситуаціях — від сезонного зберігання речей до тимчасового зберігання товарів для бізнесу. Платформа стане містком між орендарями та орендодавцями, надаючи прозорі умови оренди, зручний процес бронювання та безпечну оплату.

2.2. Опис бізнес логіки системи
1.1. Реєстрація та вхід користувача
1. Користувач може створити акаунт через електронну пошту, соціальні мережі або мобільний номер. Після реєстрації йому надається доступ до всіх функцій платформи: пошук складів, бронювання, оплата та управління орендою.
2. Якщо користувач вже зареєстрований, він може увійти в систему за допомогою свого акаунта.

1.2. Пошук складів
1. Вибір фільтрів:
Після входу в систему користувач може обрати склад за різними критеріями:
Місцезнаходження: вибір складу на карті або через список міст.
Розмір: варіанти складів різних розмірів, від малих до великих.
Ціна: можливість встановити бюджет, в межах якого користувач готовий орендувати склад.
Додаткові послуги: чи потрібен користувачу додатковий доступ, охорона, клімат-контроль, електричні розетки тощо.
2. Перегляд результатів:
Система показує список доступних складів на основі обраних фільтрів, включаючи фотографії, умови оренди, ціну та відгуки інших користувачів.

1.3. Оренда складу
1. Вибір складу та бронювання:
Користувач вибирає склад, який відповідає його вимогам, і переходить до етапу бронювання. Тут він вказує терміни оренди (дати початку та закінчення), після чого система перевіряє доступність складу на ці дати.
2. Підтвердження умов оренди:
Платформа показує деталі оренди (ціна, умови, розмір) та дозволяє користувачу підтвердити бронювання.
3. Оплата:
Після підтвердження бронювання, користувач може здійснити оплату через інтегровані платіжні системи (кредитні картки, PayPal, мобільні платежі тощо).

1.4. Доступ до складу
1. Отримання доступу до складу:
Після успішної оплати користувач отримує деталі доступу до складу — ключі, код доступу, або інструкції для використання платформи (якщо склад має автоматизовану систему доступу).
2. Контроль термінів оренди:
Користувач отримує нагадування про завершення оренди за кілька днів до закінчення терміну. Він може продовжити оренду через систему або звільнити склад вчасно.

1.5. Управління орендою
1. Перегляд історії оренди:
Користувач має доступ до історії своїх оренд, де можна побачити деталі кожного бронювання: дати, ціни, умови оренди.
2. Продаж додаткових послуг:
Користувач може вибрати додаткові послуги (наприклад, підвищення рівня безпеки, продовження оренди, зберігання вантажу тощо), що надаються орендодавцями складів.



3. Scope and Limitations
3.1. Реалізація, опис серверної частини
1. Реєстрація та автентифікація користувачів:
• Користувачі можуть реєструватися через API, вказуючи необхідні дані (ім’я, email, пароль), після чого вони отримують доступ до свого облікового запису.
• Реалізація автентифікації забезпечує безпеку та дозволяє користувачам входити в систему, використовуючи email та пароль. Можна інтегрувати OAuth для аутентифікації через соціальні мережі або Google.
• Після успішного входу користувачі отримують токен авторизації, який вони використовують для доступу до захищених ресурсів.

2. Управління складами:
• Система надає можливість адміністраторам додавати, редагувати та видаляти інформацію про доступні склади. Включає локацію, розмір, ціну та доступність складу.
• Користувачі можуть переглядати доступні склади через інтерфейс API, отримуючи детальну інформацію про кожен склад, зокрема розміри та ціну за оренду.
• Користувачі можуть фільтрувати склади за різними критеріями, такими як локація, ціна та доступність.

3. Бронювання складів:
• Користувачі можуть створювати бронювання для складів, вибираючи відповідні складські приміщення на певні дати. Система перевіряє наявність складу в обраний період та бронює його.
• Для кожного бронювання створюється запис у базі даних, що містить інформацію про користувача, склад, дати оренди та суму до сплати.
• Користувачі можуть переглядати свої бронювання, оновлювати чи скасовувати їх.

4. Оплата та транзакції:
• Для підтвердження бронювання та завершення процесу оренди користувачі здійснюють оплату через інтеграцію з платіжними системами (наприклад, Stripe або PayPal).
• Після успішної оплати система генерує квитанцію та підтверджує бронювання.
• Система зберігає інформацію про транзакції у базі даних для подальшого моніторингу та звітності.

5. Історія оренди та управління обліковим записом:
• Користувачі мають доступ до історії своїх бронювань та оплат.
• Вони можуть оновлювати свої особисті дані та переглядати деталі попередніх оренд.
• Адміністратори мають доступ до управління користувачами та можуть переглядати всю історію оренд та платежів.


Технічні деталі:
• FastAPI обробляє всі HTTP-запити, забезпечуючи високу швидкість роботи та підтримку асинхронних операцій.
• PostgreSQL використовується для зберігання даних у структурованому вигляді, що дозволяє масштабувати систему для великої кількості користувачів та складів.
• Для безпеки даних користувачів використовуються хешування паролів (наприклад, bcrypt) та токенізація (JWT) для автентифікації користувачів.
• SQLAlchemy забезпечує ORM для зручної роботи з базою даних та підтримує складні запити.


4.3  Використані технології
Backend:
• Python — основна мова програмування для серверної логіки.
• FastAPI — веб-фреймворк для створення швидких і ефективних RESTful API.
• PostgreSQL — реляційна база даних для зберігання інформації про користувачів, склади, оренди та платежі.
• SQLAlchemy — ORM для інтеграції з PostgreSQL.
• Alembic — для управління міграціями бази даних.
• JWT (JSON Web Token) — для автентифікації та авторизації користувачів через токени.
• Docker — для контейнеризації додатка та полегшення деплойменту.

Frontend (Web):
• React — бібліотека для побудови інтерфейсів користувача на фронтенді.
• React Router — для маршрутизації у додатку.
• Redux або Context API — для управління станом додатка.
• Axios або Fetch API — для взаємодії з сервером через HTTP запити.
• Material-UI або Tailwind CSS — для стилізації компонентів інтерфейсу.
• Formik або React Hook Form — для управління формами.
• React Query — для ефективного оброблення запитів і кешування даних.

Mobile :
