1. Business Requirements
<The business requirements provide the foundation and reference for all detailed requirements development. You may gather business requirements from the customer or development organization’s senior management, an executive sponsor, a project visionary, product management, the marketing department, or other individuals who have a clear sense of why the project is being undertaken and the ultimate value it will provide, both to the business and to customers.>
1.1. Background
У сучасному світі багато людей стикаються з проблемою зберігання своїх речей через обмежений простір у домі або офісі. Це може бути зумовлено переїздами, сезонними змінами або просто потребою в тимчасовому звільненні місця. Часто виникає потреба у зберіганні речей на складі, але знайти доступне і зручне місце для оренди може бути складно.
Програмна система для оренди маленьких складів пропонує рішення цієї проблеми. Вона дозволяє користувачам легко знаходити, бронювати і оплачувати склади, забезпечуючи зручний доступ до цієї послуги без зайвих труднощів. Система автоматизує процеси бронювання та управління орендою складів, що зменшує час і зусилля, необхідні для пошуку і організації зберігання речей.
1.2. Аналіз аналогічних рішень
Neighbor
Це онлайн-платформа для оренди складів, що дозволяє приватним особам здавати свої додаткові приміщення або простір (гаражі, підвали, тощо) для зберігання речей. Система дозволяє орендодавцям та орендарям безпосередньо взаємодіяти через платформу.
Переваги:
• Різноманіття варіантів складів, від приватних гаражів до великих комерційних складів.
• Низька вартість оренди завдяки використанню приватних приміщень.
• Гнучкість у виборі розміру складу та умов оренди.
Недоліки:
• Залежність від місцезнаходження доступних складів.
• Відсутність сертифікацій для приватних орендодавців, що може створювати проблеми з безпекою.

Public Storage
Це  одна з найбільших компаній, що надає послуги з оренди складів в США. Платформа дозволяє користувачам знайти, забронювати і оплатити склад через інтернет. Користувачі можуть вибирати між різними типами складів, від малих до великих.
Переваги:
• Велика кількість складів в різних регіонах.
• Високий рівень безпеки на складах.
• Наявність додаткових послуг, таких як продаж пакувальних матеріалів.
Недоліки:
• Висока вартість оренди порівняно з іншими варіантами.
• Обмежений доступ до складів у нічний час.

1.3. Монетизація
• Плата за оренду: Основне джерело доходу — комісія з кожної оренди складу. Користувачі платять за послугу оренди складу, а платформа отримує частку від цієї транзакції.
• Додаткові послуги: Можливість надання додаткових послуг, таких як охорона, клімат-контроль, пакувальні матеріали, доступ до транспорту для перевезення речей тощо. Ці послуги можуть бути надані за додаткову плату, що збільшує загальний дохід.
• Платні підвищення: Пропозиція платних підписок для орендодавців, що дозволяють отримати додаткові функції, такі як пріоритетна позиція в результатах пошуку, аналітика використання складу або більша видимість серед користувачів.
1.4. Customer or Market Needs
Клієнти потребують зручних та доступних рішень для оренди складів. Вони хочуть знайти різні варіанти складів на одній платформі, з можливістю порівняти ціни, розміри та локації. Важливою є проста та інтуїтивно зрозуміла система бронювання, яка дозволить швидко забронювати склад без зайвих складнощів. Користувачі також очікують надійної підтримки, можливості для безпечної оплати та прозорих умов оренди. Додатково важливими є зручні інтерфейси для відстеження стану оренди, доступ до послуг охорони чи клімат-контролю, а також прості способи управління орендами через мобільні пристрої та веб-платформи.

1.5. Business Risks
• Конкуренція на ринку: Існує багато платформ для оренди складів, і на ринку можуть бути вже добре розвинені конкуренти з великим досвідом та лояльною базою користувачів. Новому продукту буде важко виділитися серед них, особливо на початкових етапах.
• Невизначеність попиту: Ринок оренди маленьких складів може бути нестабільним, і попит може змінюватися залежно від економічних умов або змін у поведінці споживачів. Це може призвести до низької кількості користувачів на платформі.
• Технічні проблеми та безпека: Оскільки платформа буде залежати від технологій для обробки замовлень, оплати та управління даними користувачів, можуть виникнути технічні проблеми або загрози безпеці, які можуть вплинути на довіру користувачів та репутацію бізнесу.
• Регулювання та юридичні вимоги: Зміни в законодавстві, пов’язані з орендою нерухомості, податками або захистом даних, можуть призвести до необхідності змін у роботі платформи або додаткових витрат на забезпечення юридичних вимог.
• Низька ліквідність складів: У разі, якщо власники складів не будуть зацікавлені в розміщенні своїх об'єктів на платформі або у разі низького попиту, може виникнути проблема з поповненням асортименту та забезпеченням користувачів достатньою кількістю складів.
• Фінансові ризики: Недостатнє фінансування для розвитку платформи, маркетингових кампаній або підтримки користувачів може призвести до затримок у досягненні фінансових цілей або невдач у запуску ключових функцій.
• Залучення та утримання користувачів: Якщо платформа не зможе забезпечити зручний досвід або підтримку, користувачі можуть швидко перейти до конкурентів, що вплине на стабільність та розвиток бізнесу.
• Непередбачувані витрати: Витрати на розвиток технологій, маркетинг, партнерства та підтримку клієнтів можуть бути вищими за заплановані, що може привести до фінансових труднощів на ранніх етапах.

2.1. Vision Statement
Наша мета — створити інноваційну онлайн-платформу для оренди маленьких складів, яка стане зручним інструментом для людей та бізнесів, що потребують тимчасового зберігання речей. Ми прагнемо створити просте та доступне середовище, в якому кожен користувач, від індивідуального клієнта до малого бізнесу, може без зайвих труднощів орендувати склад, отримуючи зручний доступ до необхідних послуг і швидку підтримку.
Наша система буде надавати доступ до широкого асортименту складських приміщень різних розмірів і умов, щоб задовольнити потреби користувачів у різних ситуаціях — від сезонного зберігання речей до тимчасового зберігання товарів для бізнесу. Платформа стане містком між орендарями та орендодавцями, надаючи прозорі умови оренди, зручний процес бронювання та безпечну оплату.
Збалансована стратегія розвитку платформи спрямована на задоволення потреб різних сегментів ринку, включаючи фізичних осіб, малі та середні бізнеси, а також орендодавців. Ми віримо, що наш продукт значно спростить процес оренди складських приміщень, полегшить доступ до необхідних послуг і забезпечить високий рівень задоволення від користування.

2.2. Major Features
1.1. Реєстрація та вхід користувача
1. 
Користувач може створити акаунт через електронну пошту, соціальні мережі або мобільний номер. Після реєстрації йому надається доступ до всіх функцій платформи: пошук складів, бронювання, оплата та управління орендою.
2. Вхід до акаунту:
Якщо користувач вже зареєстрований, він може увійти в систему за допомогою свого акаунта.

1.2. Пошук складів
1. Вибір фільтрів:
Після входу в систему користувач може обрати склад за різними критеріями:
o Місцезнаходження: вибір складу на карті або через список міст.
o Розмір: варіанти складів різних розмірів, від малих до великих.
o Ціна: можливість встановити бюджет, в межах якого користувач готовий орендувати склад.
o Додаткові послуги: чи потрібен користувачу додатковий доступ, охорона, клімат-контроль, електричні розетки тощо.
2. Перегляд результатів:
Система показує список доступних складів на основі обраних фільтрів, включаючи фотографії, умови оренди, ціну та відгуки інших користувачів.

1.3. Оренда складу
1. Вибір складу та бронювання:
Користувач вибирає склад, який відповідає його вимогам, і переходить до етапу бронювання. Тут він вказує терміни оренди (дати початку та закінчення), після чого система перевіряє доступність складу на ці дати.
2. Підтвердження умов оренди:
Платформа показує деталі оренди (ціна, умови, розмір) та дозволяє користувачу підтвердити бронювання.
3. Оплата:
Після підтвердження бронювання, користувач може здійснити оплату через інтегровані платіжні системи (кредитні картки, PayPal, мобільні платежі тощо).

1.4. Доступ до складу
1. Отримання доступу до складу:
Після успішної оплати користувач отримує деталі доступу до складу — ключі, код доступу, або інструкції для використання платформи (якщо склад має автоматизовану систему доступу).
2. Контроль термінів оренди:
Користувач отримує нагадування про завершення оренди за кілька днів до закінчення терміну. Він може продовжити оренду через систему або звільнити склад вчасно.

1.5. Управління орендою
1. Перегляд історії оренди:
Користувач має доступ до історії своїх оренд, де можна побачити деталі кожного бронювання: дати, ціни, умови оренди.
2. Продаж додаткових послуг:
Користувач може вибрати додаткові послуги (наприклад, підвищення рівня безпеки, продовження оренди, зберігання вантажу тощо), що надаються орендодавцями складів.

1.6. Взаємодія з орендодавцями
1. Залишення відгуків:
Після завершення оренди, користувач може залишити відгук про склад і орендодавця. Це допомагає майбутнім користувачам зробити обґрунтований вибір.
2. Запит на спеціальні умови:
Якщо користувач має особливі вимоги до складу (наприклад, тривала оренда, додаткові послуги), він може зв'язатися з орендодавцем через внутрішній чат або систему запитів.

1.7. Безпека та конфіденційність
1. Захист персональних даних:
Платформа забезпечує надійне зберігання персональних даних користувача, використовуючи сучасні методи шифрування.
2. Безпека транзакцій:
Усі фінансові операції проводяться через захищені платіжні шлюзи, що гарантують безпеку платежів.

1.8. Підтримка користувачів
1. Технічна підтримка:
У разі виникнення проблем користувач може звернутися до служби підтримки через чат або електронну пошту.
2. FAQ та допомога:
Платформа має розділ з часто задаваними питаннями, який допоможе користувачам знайти відповіді на основні запитання щодо оренди, доступу та оплати.

2.3. Assumptions and Dependencies
• Інтеграція з платіжними системами: Припускається, що платіжні системи, такі як Stripe, PayPal або локальні платіжні провайдери, будуть успішно інтегровані в платформу для забезпечення безпечних і зручних транзакцій для користувачів.
• Доступність складів: Очікується, що в різних регіонах буде достатня кількість складів для оренди, що забезпечить широке покриття та задоволення попиту користувачів.
• Інтернет-з'єднання: Для користування платформою припускається наявність стабільного доступу до Інтернету на мобільних пристроях або стаціонарних комп'ютерах, що дозволить користувачам безперешкодно взаємодіяти з платформою.
• Технічні ресурси: Припускається, що команда розробників матиме необхідні ресурси для підтримки і розвитку платформи, включаючи сервери, хостинг та API для інтеграцій з третіми сторонами, як-от служби доставки або платіжні системи.
• Попит на оренду складів: Очікується, що попит на оренду складів для зберігання речей буде стабільним і зростаючим, зокрема в міських районах і в періоди сезонних змін або переїздів.
• Регуляторні вимоги: Припускається, що платформа буде відповідати всім юридичним вимогам щодо оренди нерухомості, захисту даних та обробки платежів в різних країнах, де вона буде працювати.
• Співпраця з логістичними компаніями: Очікується, що будуть встановлені партнерські відносини з логістичними компаніями для забезпечення ефективних і швидких послуг доставки складів та додаткових послуг для користувачів.

Clarifying the scope and limitations helps to establish realistic expectations of the many stakeholders. It also provides a reference frame against which proposed features and requirements changes can be evaluated. 3.1. Реалізація, опис серверної частини
1. Реєстрація та автентифікація користувачів:
o Користувачі можуть реєструватися через API, вказуючи необхідні дані (ім’я, email, пароль), після чого вони отримують доступ до свого облікового запису.
o Реалізація автентифікації забезпечує безпеку та дозволяє користувачам входити в систему, використовуючи email та пароль. Можна інтегрувати OAuth для аутентифікації через соціальні мережі або Google.
o Після успішного входу користувачі отримують токен авторизації, який вони використовують для доступу до захищених ресурсів.
2. Управління складами:
o Система надає можливість адміністраторам додавати, редагувати та видаляти інформацію про доступні склади. Включає локацію, розмір, ціну та доступність складу.
o Користувачі можуть переглядати доступні склади через інтерфейс API, отримуючи детальну інформацію про кожен склад, зокрема розміри та ціну за оренду.
o Користувачі можуть фільтрувати склади за різними критеріями, такими як локація, ціна та доступність.
3. Бронювання складів:
o Користувачі можуть створювати бронювання для складів, вибираючи відповідні складські приміщення на певні дати. Система перевіряє наявність складу в обраний період та бронює його.
o Для кожного бронювання створюється запис у базі даних, що містить інформацію про користувача, склад, дати оренди та суму до сплати.
o Користувачі можуть переглядати свої бронювання, оновлювати чи скасовувати їх.
4. Оплата та транзакції:
o Для підтвердження бронювання та завершення процесу оренди користувачі здійснюють оплату через інтеграцію з платіжними системами (наприклад, Stripe або PayPal).
o Після успішної оплати система генерує квитанцію та підтверджує бронювання.
o Система зберігає інформацію про транзакції у базі даних для подальшого моніторингу та звітності.
5. Історія оренди та управління обліковим записом:
o Користувачі мають доступ до історії своїх бронювань та оплат.
o Вони можуть оновлювати свої особисті дані та переглядати деталі попередніх оренд.
o Адміністратори мають доступ до управління користувачами та можуть переглядати всю історію оренд та платежів.
6. API та інтеграція з іншими системами:
o Система надає RESTful API для інтеграції з іншими сервісами та для роботи з мобільними або веб-клієнтами.
o Інтерфейси API включають доступ до інформації про склади, бронювання, користувачів та платежі.
o Для забезпечення високої продуктивності та зручності використовується кешування та оптимізація запитів до бази даних.
o 
Технічні деталі:
• FastAPI обробляє всі HTTP-запити, забезпечуючи високу швидкість роботи та підтримку асинхронних операцій.
• PostgreSQL використовується для зберігання даних у структурованому вигляді, що дозволяє масштабувати систему для великої кількості користувачів та складів.
• Для безпеки даних користувачів використовуються хешування паролів (наприклад, bcrypt) та токенізація (JWT) для автентифікації користувачів.
• SQLAlchemy забезпечує ORM для зручної роботи з базою даних та підтримує складні запити.

4.3. Використані технології
Backend:
• Python — основна мова програмування для серверної логіки.
• FastAPI — веб-фреймворк для створення швидких і ефективних RESTful API.
• PostgreSQL — реляційна база даних для зберігання інформації про користувачів, склади, оренди та платежі.
• SQLAlchemy — ORM для інтеграції з PostgreSQL.
• Alembic — для управління міграціями бази даних.
• JWT (JSON Web Token) — для автентифікації та авторизації користувачів через токени.
• Docker — для контейнеризації додатка та полегшення деплойменту.

Frontend (Web):
• React — бібліотека для побудови інтерфейсів користувача на фронтенді.
• React Router — для маршрутизації у додатку.
• Redux або Context API — для управління станом додатка.
• Axios або Fetch API — для взаємодії з сервером через HTTP запити.
• Material-UI або Tailwind CSS — для стилізації компонентів інтерфейсу.
• Formik або React Hook Form — для управління формами.
• React Query — для ефективного оброблення запитів і кешування даних.

Mobile (React Native):
• React Native — для створення мобільного додатку для iOS і Android з одним кодом.
• React Navigation — для навігації між екранами в мобільному додатку.
• Axios або Fetch API — для взаємодії з сервером через HTTP запити.
• Redux або Context API — для управління станом додатка на мобільному пристрої.
